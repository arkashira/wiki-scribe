# ROADMAP.md
## Introduction
The wiki-scribe project aims to create a wiki-building platform that leverages Large Language Models (LLMs) to generate high-quality, spoiler-free content, while automating quality control and editing processes for solo developers and fan-wiki operators. This roadmap outlines the key milestones and phases for the project, ensuring a clear path to launch and beyond.

## MVP Milestone (Must-Have for Launch)
The following features are critical for the Minimum Viable Product (MVP) launch:
### Core Features
* **LLM Integration**: Integrate a verified LLM (e.g., vLLM) for content generation
* **Spoiler-Free Content**: Develop and implement algorithms to ensure generated content is spoiler-free
* **Automated Quality Control**: Design and implement automated quality control checks for generated content
* **Basic Editing Interface**: Create a basic editing interface for users to review and modify generated content
* **User Authentication**: Implement user authentication for access control and content management

## v1 Phase (Post-Launch)
The v1 phase will focus on enhancing the core features, improving user experience, and expanding the platform's capabilities.
### Themes
* **Content Expansion**: Introduce new content types (e.g., images, videos) and improve LLM training data
* **User Experience**: Enhance the editing interface, add features for user collaboration, and improve overall usability
* **Quality Control**: Introduce human-in-the-loop quality control processes and refine automated checks
* **Integration and API**: Develop APIs for integrating wiki-scribe with other platforms and services

### Key Features
* **Image and Video Support**: Allow users to add images and videos to wiki pages
* **Collaboration Tools**: Introduce features for real-time collaboration, commenting, and @mentions
* **Customizable Quality Control**: Enable users to define custom quality control checks and workflows
* **API and Webhooks**: Expose APIs for integrating wiki-scribe with other services and provide webhooks for event notifications

## v2 Phase (Growth and Expansion)
The v2 phase will focus on scaling the platform, improving performance, and introducing new features to increase user engagement and retention.
### Themes
* **Scalability and Performance**: Optimize the platform for large-scale usage, improve response times, and ensure high availability
* **User Engagement**: Introduce features to increase user engagement, such as gamification, rewards, and social sharing
* **Content Discovery**: Develop algorithms and features to help users discover new and relevant content
* **Enterprise Features**: Introduce features for large-scale enterprises, such as single sign-on, custom branding, and advanced analytics

### Key Features
* **Gamification and Rewards**: Introduce a points system, badges, and rewards for user contributions and engagement
* **Content Recommendations**: Develop algorithms to recommend relevant content to users based on their interests and behavior
* **Single Sign-On**: Support single sign-on for large-scale enterprises and organizations
* **Advanced Analytics**: Provide detailed analytics and insights for enterprise users to track user engagement and content performance

By following this roadmap, the wiki-scribe project will deliver a robust and feature-rich platform for solo developers and fan-wiki operators, while continuously improving and expanding its capabilities to meet the evolving needs of its users.
